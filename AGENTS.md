# AGENTS.md

## Run And Verify
- `npm run dev` starts the Next.js app on `http://localhost:9002` with Turbopack.
- Match CI before considering work done: `npm run typecheck && npm run lint && npm run build`.
- `npm run build` is not a full quality gate here: `next.config.ts` sets `typescript.ignoreBuildErrors` and `eslint.ignoreDuringBuilds`, so build success can hide broken TS or ESLint.
- There is no checked-in test suite or `test` script right now; focused verification is typecheck, lint, and build.
- Genkit work is separate from the site: use `npm run genkit:dev` or `npm run genkit:watch` to load `src/ai/dev.ts`.

## Structure
- This is a single-route App Router site. `src/app/page.tsx` is a client component that composes the whole landing page and controls the hash-driven `DetailDrawer`.
- Most resume content is data-driven from `src/data/*.json` through `src/lib/data.ts`; prefer editing data files before changing section component markup.
- User-facing copy is centralized in `src/lib/i18n.ts` for `en`, `es`, and `pt`. When changing text, update all locales together.
- Language selection is client-side only and persisted in `localStorage` under `portfolio-locale` (`src/components/context/LanguageContext.tsx`).
- The contact form is a static-site `mailto:` flow in `src/components/sections/Contact.tsx`; there is no existing backend form handler.

## Static Export And Deploy
- Production is a GitHub Pages static export. `next.config.ts` uses `output: 'export'`, `trailingSlash: true`, and production-only `basePath`/`assetPrefix` derived from the repo name `Eduardo_Sacahui_Resume`.
- If the repo name or public site URL changes, update both `next.config.ts` and `.github/workflows/deploy.yml`; metadata also reads `NEXT_PUBLIC_SITE_URL` in `src/app/layout.tsx`, `src/app/sitemap.ts`, and `src/app/robots.ts`.
- GitHub Actions is the executable deploy source of truth: it installs with npm, runs `typecheck -> lint -> build`, exports `out/`, and adds `out/.nojekyll` before Pages deploy.

## Assets
- The downloadable CV at `public/Eduardo_Sacahui_Resume.pdf` is generated by `python3 scripts/generate_cv_pdf.py`. If resume data changes and the PDF should stay in sync, regenerate it manually.
